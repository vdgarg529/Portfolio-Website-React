# Encapsulation in C++

## 1. Detailed Notes

### What is Encapsulation?
Encapsulation is the **OOP principle** of **wrapping data (attributes) and methods (functions)** together into a single unit (class).  
It restricts **direct access** to data and ensures controlled access using **getters** and **setters**.

Encapsulation = **Data Hiding + Data Binding**

- **The "Bundling" Part (Data Binding):** A class combines data (member variables or attributes) and functions (member functions or methods) together.
    
- **The "Access Control (Data Hiding)" Part:** This is implemented using **access specifiers**: `public`, `private`, and `protected`.

### Why it's important:
- Encapsulation enforces data hiding and ensures that an object's internal state can only be changed in controlled ways, typically through public member functions (often called **getters** and **setters**). This increases security, prevents data corruption, and makes the code easier to maintain and reason about.



## 2. Definition
- **Encapsulation**: The process of **binding data members and methods together** and restricting direct access to some of the object’s components.

---

## 3. Key Points to Remember
- Achieved by declaring attributes as **private/protected**.  
- Access provided through **public getter and setter methods**.  
- Ensures **data security** and prevents accidental misuse.  
- Promotes **maintainability and flexibility**.

## 5. C++ Implementation

```cpp
class BankAccount {
private: // Data is "hidden"
    double balance;
    string accountNumber;

public: // Public interface to interact with the data
    // Constructor
    BankAccount(double initialBalance, string accNum) {
        balance = initialBalance;
        accountNumber = accNum;
    }

    // Getter function (provides read-only access)
    double getBalance() const {
        return balance;
    }

    // Setter function (provides controlled modification)
    void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        }
    }
};

int main() {
    BankAccount myAccount(1000.0, "ACC123");
    // myAccount.balance = 1000000; // ERROR: 'balance' is private
    myAccount.deposit(500.0); // Correct and safe way to modify balance
    cout << myAccount.getBalance(); // Correct way to read balance
    return 0;
}
}
```


## 6. Real World Example

Think of **Encapsulation** like a **capsule of medicine**:

- The **medicine (data)** is hidden inside.
    
- You can **use it only through the capsule (class methods)**, not by directly accessing the powder inside.
    

Similarly, in an ATM machine:

- **Balance** is private.
    
- You access it only through functions like **withdraw()**, **deposit()**, not directly.

### Theory

1. **Q:** What is encapsulation in C++?  
    **A:** Encapsulation is the process of **binding data and methods** into a single unit and restricting direct access to data.
    
2. **Q:** How is encapsulation achieved in C++?  
    **A:** By declaring data members as **private/protected** and using **public getter/setter methods**.

    **A:** `private`.
    
4. **Q:** Difference between encapsulation and abstraction?  
    **A:**
    
    - Encapsulation = **Hiding data** (data security).
        
    - Abstraction = **Hiding implementation** (conceptual hiding).
        
5. **Q:** Why is encapsulation important?  
    **A:** It ensures **data security, modularity, maintainability**, and prevents misuse of data.

8. **Q:** Show an example where encapsulation prevents invalid data entry (like negative age).
    

```cpp
#include <iostream>
using namespace std;

class Person {
private:
    int age;

public:
    void setAge(int a) {
        if (a >= 0)
            age = a;
        else
            cout << "Invalid age!" << endl;
    }

    int getAge() {
        return age;
    }
};

int main() {
    Person p;
    p.setAge(-5);  // rejected
    p.setAge(25);  // valid
    cout << "Age: " << p.getAge() << endl;
    return 0;
}

```

## 8. Multiple Choice Questions (MCQs)

1. Encapsulation in C++ is achieved using:  
    a) Classes and Objects  
    b) Inheritance  
    c) Functions only  
    d) Polymorphism  
    **Answer:** a) Classes and Objects
    
2. Which access modifier is mostly used for encapsulation?  
    a) public  
    b) private  
    c) protected  
    d) friend  
    **Answer:** b) private
    
3. Encapsulation is also called:  
    a) Data Binding  
    b) Data Abstraction  
    c) Data Inheritance  
    d) Data Conversion  
    **Answer:** a) Data Binding
    
4. What does encapsulation primarily provide?  
    a) Speed optimization  
    b) Data security  
    c) More memory usage  
    d) Faster compilation  
    **Answer:** b) Data security
    
5. In a class, data members are usually:  
    a) public  
    b) private  
    c) protected  
    d) static  
    **Answer:** b) private
    
6. Getters and setters are used to:  
    a) Hide data  
    b) Access private data  
    c) Delete private members  
    d) Create classes  
    **Answer:** b) Access private data
    
7. If a data member is declared private:  
    a) Accessible everywhere  
    b) Accessible only within the class  
    c) Accessible in derived classes  
    d) Accessible in main()  
    **Answer:** b) Accessible only within the class
    
8. Which of these best describes encapsulation?  
    a) Hiding the implementation details  
    b) Binding data and functions together  
    c) Reusing code  
    d) Overloading operators  
    **Answer:** b) Binding data and functions together
    
9. Which principle is violated if all members are public?  
    a) Abstraction  
    b) Encapsulation  
    c) Polymorphism  
    d) Inheritance  
    **Answer:** b) Encapsulation
    
10. Encapsulation is mainly implemented in C++ using:  
    a) `public`  
    b) `private` and `protected`  
    c) `friend`  
    d) `extern`  
    **Answer:** b) private and protected
