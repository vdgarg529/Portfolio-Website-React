# Inheritance in C++

## 1. Detailed Notes

### Introduction
- **Inheritance** is an OOP mechanism where a new class (**derived/child**) acquires properties and behaviors of an existing class (**base/parent**).  
- Promotes **code reusability**.  
- Derived class can **extend or override** parent functionality.  

---

### Code Reusability
- Base class defines common attributes/methods.  
- Derived classes reuse them without rewriting.  

---

### Constructor and Destructor Call Order
1. **Constructor Call Order**  
   - Parent (base class) constructor is called **first**.  
   - Then child (derived class) constructor executes.  

2. **Destructor Call Order**  
   - Child destructor executes **first**.  
   - Then parent destructor is called.  

üëâ This ensures resources are allocated from top to bottom, but released from bottom to top.

---

### Default Constructor of Parent Class
- If no constructor is explicitly called in derived class, the **default constructor of parent class** is invoked automatically.  

---

### Parameterized Constructor in Inheritance
- To call a **parameterized constructor of parent**, derived class must use **initializer list syntax**.

```cpp
Derived(int x, int y) : Base(x) {
    // derived class body
}


```

### Mode of Inheritance

Inheritance can be declared with **access specifiers**:

1. **Public Inheritance**
    
    - public in parent ‚Üí public in child
        
    - protected in parent ‚Üí protected in child
        
    - private in parent ‚Üí inaccessible
        
2. **Protected Inheritance**
    
    - public in parent ‚Üí protected in child
        
    - protected in parent ‚Üí protected in child
        
3. **Private Inheritance**
    
    - public in parent ‚Üí private in child
        
    - protected in parent ‚Üí private in child
        

---

### Types of Inheritance

1. **Single Inheritance** ‚Äì One parent, one child.
    
2. **Multiple Inheritance** ‚Äì Child class inherits from multiple parents.
    
3. **Multilevel Inheritance** ‚Äì Inheritance across multiple levels.
    
4. **Hierarchical Inheritance** ‚Äì One parent, multiple children.
    
5. **Hybrid Inheritance** ‚Äì Combination of above (may involve ambiguity).
    

---

## 2. Definitions

- **Inheritance:** Mechanism of creating new classes from existing ones.
    
- **Base Class:** The parent class.
    
- **Derived Class:** The child class.
    
- **Mode of Inheritance:** The access control (public/protected/private) used when inheriting.
    

---

## 3. Key Points to Remember

- Base constructor executes **before** derived.
    
- Derived destructor executes **before** base destructor.
    
- If not specified, base **default constructor** is automatically called.
    
- Parameterized base constructor must be explicitly called in derived class.
    
- **Public inheritance** = ‚Äúis-a‚Äù relationship.

## 5. C++ Implementation

### Basic Inheritance

```cpp
#include <iostream>
using namespace std;

class Animal {
public:
    void eat() { cout << "Eating..." << endl; }
};

class Dog : public Animal {
public:
    void bark() { cout << "Barking..." << endl; }
};

int main() {
    Dog d;
    d.eat();   // inherited
    d.bark();  // child-specific
    return 0;
}

```

---

### Constructor and Destructor Order

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    Base() { cout << "Base constructor\n"; }
    ~Base() { cout << "Base destructor\n"; }
};

class Derived : public Base {
public:
    Derived() { cout << "Derived constructor\n"; }
    ~Derived() { cout << "Derived destructor\n"; }
};

int main() {
    Derived d;
    return 0;
}

```

### Parameterized Constructor in Inheritance


## 6. Real World Example

- **Inheritance:** A **Car** class inherits from **Vehicle**.
    
    - Vehicle defines common methods: `start()`, `stop()`.
        
    - Car adds new methods: `airConditioner()`.
        
- **Code Reuse:** Every vehicle doesn‚Äôt need to rewrite `start()`.
    
- **Constructor Order:** When building a car, **chassis (base)** is built before the **body (derived)**, but when dismantling, body is removed first.

## 7. Questions with Answers

### Theory

1. **Q:** What is inheritance?  
    **A:** It‚Äôs the mechanism of deriving a new class (child) from an existing class (parent), enabling code reuse.
    
2. **Q:** Which constructor is called first in inheritance?  
    **A:** Base class constructor is called first, then derived class constructor.
    
3. **Q:** Which destructor is called first in inheritance?  
    **A:** Derived class destructor is called first, then base class destructor.
    
4. **Q:** What happens if base class has no default constructor but only a parameterized one?  
    **A:** Derived class must explicitly call the base parameterized constructor.
    
5. **Q:** Difference between public, protected, and private inheritance?  
    **A:**
    
    - Public ‚Üí base public ‚Üí child public
        
    - Protected ‚Üí base public ‚Üí child protected
        
    - Private ‚Üí base public/protected ‚Üí child private

## 8. Multiple Choice Questions (MCQs)

1. In inheritance, which constructor is called first?  
    a) Derived  
    b) Base  
    c) Both simultaneously  
    d) None  
    **Answer:** b) Base
    
2. Which destructor is called first?  
    a) Base  
    b) Derived  
    c) Both simultaneously  
    d) None  
    **Answer:** b) Derived
    
3. If a derived class constructor does not explicitly call base constructor:  
    a) Base constructor is not called  
    b) Base default constructor is called automatically  
    c) Compiler error  
    d) None  
    **Answer:** b) Base default constructor is called automatically
    
4. Public inheritance represents which relationship?  
    a) Has-a  
    b) Is-a  
    c) Uses-a  
    d) None  
    **Answer:** b) Is-a
    
5. In protected inheritance, public members of base become:  
    a) public  
    b) protected  
    c) private  
    d) not accessible  
    **Answer:** b) protected
    
6. In private inheritance, protected members of base become:  
    a) public  
    b) protected  
    c) private  
    d) not accessible  
    **Answer:** c) private
    
7. Which type of inheritance can cause the Diamond Problem?  
    a) Single  
    b) Multiple  
    c) Multilevel  
    d) Hierarchical  
    **Answer:** b) Multiple
    
8. Which inheritance involves one parent and multiple children?  
    a) Single  
    b) Multiple  
    c) Hierarchical  
    d) Hybrid  
    **Answer:** c) Hierarchical
    
9. Which inheritance involves parent ‚Üí child ‚Üí grandchild?  
    a) Multilevel  
    b) Multiple  
    c) Hybrid  
    d) Hierarchical  
    **Answer:** a) Multilevel
    
10. Which mode of inheritance allows base public members to remain public in child?  
    a) Private  
    b) Protected  
    c) Public  
    d) Virtual  
    **Answer:** c) Public
