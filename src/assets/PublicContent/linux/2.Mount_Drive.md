## Method 1 — Mount via Terminal (Manual)

### Step 1 — Find the drive

Open terminal:

```bash
lsblk
```

You’ll see something like:

```bash
sda      1TB
├─sda1
sdb      500G   ← your new SSD/HDD
└─sdb1
```


---

### Step 2 — Create mount point

```bash
sudo mkdir /mnt/mydrive
```


---

### Step 3 — Mount it

#### For EXT4

```bash
sudo mount /dev/sdb1 /mnt/mydrive
```


#### For NTFS

```bash
sudo mount -t ntfs-3g /dev/sda2 /mnt/sda2
```


Replace:

- `/dev/sdb1` → your partition
    
- `/mnt/mydrive` → your folder





## Method 2 — Permanent Mount (Auto-mount on boot)

### Step 1 — Get UUID

```bash
sudo blkid
```

Copy the UUID of your partition.

---

### Step 2 — Edit fstab

```bash
sudo nano /etc/fstab
```

Add this line:
#### For EXT4

```bash
UUID=XXXX-XXXX  /mnt/mydrive  ext4  defaults  0  2
```


##### For NTFS

```bash
sudo apt install ntfs-3g
```


```
UUID=XXXX /mnt/sda2 ntfs-3g defaults 0 0
```

Use filesystem type accordingly:

|Filesystem|Type to write|
|---|---|
|ext4|ext4|
|NTFS|ntfs-3g|
|FAT32|vfat|

Save → reboot → drive auto-mounts.

---

## If the drive is NEW (not formatted)

Use disk tool:

```bash
sudo fdisk -l
```

Or GUI:
### Disks utility

Search **Disks** in Ubuntu → format → create partition → mount.

---

## Common Issues

### Permission issue

```bash
sudo chown -R $USER:$USER /mnt/mydrive
```




## Verify Mount

```bash
df -h
```

or

```bash
ls /mnt/sda2
```


You should see the files.
